pub const unsafe fn raw_copy(&self) -> Self

Returns a raw copy of the underlying handle pointer.

ยงSafety

As the name implies, raw_copy returns a raw copy of the handle, so closing one of the copies wonโt close the others. This means a handle can be used after it has been closed, what can lead to errors and undefined behavior. Even worse: sometimes Windows reuses handle values, so you can call a method on a completely different handle type, what can be catastrophic.

However, in some cases the Windows API demands a copy of the handle โ raw_copy is an escape hatch to fill this gap.

ยงExamples

const IDD_FOOT_ABOUTBOX: u16 = 110;

    w::IdStr::Id(IDD_HAND_ABOUTBOX),
    w::RtStr::Rt(co::RT::DIALOG),

    w::RtStr::Rt(co::RT::DIALOG),
    w::IdStr::Id(IDD_FOOT_ABOUTBOX),
    w::LANGID::NEUTRAL,
    hres_slice_lock,
)?;
